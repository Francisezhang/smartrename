# macOS 工具集 GUI 迭代规划方案

> 版本: 1.0 | 日期: 2026-04-25 | 状态: 规划中

## 执行摘要

本文档概述将6款CLI工具（smartrename、imgcrush、clipstack、dirsnap、envguard、quickmd）演进为跨平台GUI应用的策略规划。目标是扩展市场覆盖范围，从开发者用户扩展到普通macOS/Windows/Linux用户，同时保持CLI作为核心后端。

---

## 1. 市场分析

### 当前状态

- **目标用户**: 开发者、熟悉命令行的用户
- **入门门槛**: 需要命令行知识
- **市场规模**: 仅限于终端用户

### GUI 机会

| 用户群 | CLI覆盖率 | GUI覆盖率 | 增长潜力 |
|--------|----------|----------|---------|
| 开发者 | ✅ 高 | ✅ 中 | +20% |
| 设计师 | ❌ 低 | ✅ 高 | +300% |
| 内容创作者 | ❌ 低 | ✅ 高 | +500% |
| 普通Mac用户 | ❌ 无 | ✅ 高 | +1000% |

**预期影响**: GUI可将用户基数扩大3-5倍。

## 2. 技术策略

### 框架选型

**首选方案: PySide6 (Qt for Python)**

| 评估标准 | PySide6 | Tkinter | CustomTkinter | Electron |
|---------|---------|---------|---------------|----------|
| 跨平台 | ✅ 优秀 | ✅ 良好 | ✅ 良好 | ✅ 优秀 |
| 原生外观 | ✅ 是 | ❌ 否 | ⚠️ 部分 | ❌ 否 |
| 性能 | ✅ 快 | ✅ 快 | ✅ 快 | ❌ 重 |
| Python原生 | ✅ 是 | ✅ 是 | ✅ 是 | ❌ 否 |
| 富控件 | ✅ 丰富 | ❌ 基础 | ⚠️ 中等 | ✅ 丰富 |
| 打包体积 | ⚠️ 50MB | ✅ <5MB | ✅ <5MB | ❌ 150MB+ |
| 学习曲线 | ⚠️ 中等 | ✅ 低 | ✅ 低 | ⚠️ 中等 |

**推荐**: **PySide6 (Qt)** 用于专业外观和跨平台一致性。

**MVP替代方案**: **CustomTkinter** 用于快速原型开发（2周MVP）。

### 架构设计

```
┌─────────────────────────────────────────────────┐
│                  GUI 应用层                      │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────┐ │
│  │  主窗口     │  │  工具面板   │  │ 设置页   │ │
│  └─────────────┘  └─────────────┘  └──────────┘ │
│         │                │                │     │
│         └────────────────┼────────────────┘     │
│                          ▼                       │
│  ┌─────────────────────────────────────────────┐│
│  │           CLI 后端（现有代码）              ││
│  │  smartrename │ imgcrush │ clipstack │ ...   ││
│  └─────────────────────────────────────────────┘│
└─────────────────────────────────────────────────┘
```

**原则**: GUI是CLI逻辑的**包装层**。无代码重复。

### 实现方式示例

```python
# GUI调用CLI后端示例
from smartrename.core.renamer import rename_directory, preview_rename

class SmartRenameGUI:
    def preview_clicked(self):
        files = collect_files(self.directory)
        operations = preview_rename(files, self.mode)
        self.display_preview(operations)
    
    def execute_clicked(self):
        result = rename_directory(self.directory, self.mode)
        self.show_result(result)
```

## 3. 实施路线图

### 第一阶段: MVP (4周)

**目标**: 单一统一GUI启动器，基本功能。

**包含工具**（优先级）:
1. smartrename（需求最高）
2. imgcrush（可视化反馈）
3. dirsnap（树形可视化）

**交付物**:
- 统一启动器应用
- 拖拽文件选择
- 执行前预览
- 进度条
- 基础设置

### 第二阶段: 独立工具 (8周)

**目标**: 每款工具独立应用。

**第5-6周**: smartrename GUI
- 文件列表视图（复选框）
- 6种重命名模式下拉菜单
- 实时预览面板
- 撤销按钮

**第7-8周**: imgcrush GUI
- 图片预览网格
- 质量滑块
- 格式转换下拉
- 前后大小对比

**第9-10周**: clipstack GUI
- 剪贴板历史列表
- 搜索栏
- 置顶开关
- 每条记录复制按钮

**第11-12周**: dirsnap、envguard、quickmd GUI
- dirsnap: 树形视图控件，导出选项
- envguard: 密钥库列表，添加/取回对话框
- quickmd: Markdown编辑器预览

### 第三阶段: Pro功能 (4周)

**目标**: 高级GUI功能。

**功能**:
- 深色模式
- 自定义主题
- 键盘快捷键
- 批量操作队列
- 导出报告
- 历史管理
- 插件系统（未来）

### 第四阶段: 分发 (2周)

**目标**: 非开发者用户轻松安装。

**平台**:
- macOS: .app 包（PyInstaller）
- Windows: .exe 安装程序
- Linux: .deb/.rpm 包

## 7. 风险分析

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| PySide6打包体积大 | 下载慢 | 提供网络安装器 |
| 平台特性差异 | Bug | 全平台全面测试 |
| 性能问题 | 老硬件慢 | 优化，提供轻量模式 |
| 设计质量差 | UX差 | 聘请UI设计师或使用模板 |
| 维护负担重 | 两套代码 | 保持GUI薄层，复用CLI逻辑 |
| 用户困惑 | CLI vs GUI | 清晰文档，统一品牌 |
